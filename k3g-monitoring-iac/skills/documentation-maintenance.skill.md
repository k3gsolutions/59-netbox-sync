# Skill — Documentation Maintenance

## Objetivo
Manter documentação concisa, atualizada e útil para IA e equipe.

## Quando usar
- Após concluir tarefas relevantes.
- Durante revisões semanais.
- Ao detectar desatualizações.

## Entrada esperada
- Lista de mudanças realizadas.
- Arquivos impactados.
- Decisões tomadas.

## Saída esperada
- Documentação atualizada.
- CHANGELOG e baseline revisados.
- Próximas ações ajustadas.

## Checklist
- [ ] `context/CURRENT_STATE.md` atualizado?
- [ ] `context/NEXT_ACTIONS.md` alinhado?
- [ ] `CHANGELOG.md` revisado?
- [ ] `PHASE0_BASELINE.md` checkboxes atualizados?
- [ ] ADR criada/atualizada se necessário?
- [ ] Arquivos mantidos curtos e objetivos?

## Anti-padrões
- Deixar contexto obsoleto.
- Criar documentos redundantes.
- Escrever textos longos sem necessidade.
