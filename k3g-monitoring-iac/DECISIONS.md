# DECISIONS

## Visão geral
As decisões arquiteturais relevantes são registradas como ADRs em `docs/adr/`.

## Índice atual
- ADR 0001 — NetBox como single source of truth técnico.
- ADR 0002 — N8N como orquestrador principal.
- ADR 0003 — Naming e descrições machine-parseable.
- ADR 0004 — GitOps para ativos de monitoramento.
- ADR 0005 — Equipamentos read-only nas fases iniciais.

## Política
1. Toda mudança arquitetural relevante deve gerar uma ADR ou atualizar existente.
2. ADRs devem citar contexto, decisão, consequências e referências.
3. Estados: Proposto → Aceito → Superseded (quando aplicável).
