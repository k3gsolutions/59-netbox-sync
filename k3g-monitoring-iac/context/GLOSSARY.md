# GLOSSARY

- **SoT (Source of Truth)**: Sistema responsável pelos dados oficiais.
- **GitOps**: Prática de usar Git como fonte declarativa de infraestrutura/monitoramento.
- **DLQ (Dead Letter Queue)**: Fila de mensagens não processadas.
- **Dry-run**: Execução sem efeitos colaterais para validar automação.
- **ADR**: Architecture Decision Record.
- **Brownfield**: Ambiente existente já em produção.
- **Staging**: Ambiente controlado pré-produção para validação.
- **Compliance**: Aderência entre o estado desejado (NetBox/Git) e o estado aplicado (dispositivos/monitoramento).
- **Taxonomia**: Conjunto de classificações padronizadas (tags, naming).
- **DLQ Workflow**: Fluxo que consome mensagens de falha para reprocessar ou notificar.
