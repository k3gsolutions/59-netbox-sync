# Web UI PT-BR UX Copy

## Objetivo
Deixar a Web UI mais clara para operadores, NOC e engenharia, sem perder precisão técnica.

## Diretriz
- Usar PT-BR direto, curto e operacional.
- Manter termos técnicos conhecidos: NetBox, BGP, ASN, VRF, VLAN, IP, CSV, JSON, UAT.
- Traduzir apenas o texto exibido.
- Não alterar rotas, nomes de scripts, chaves internas ou enums.

## Termos sugeridos
- Outreach -> Comunicação com Times
- Service Engagement -> Pendências de Serviço
- Response / Responses -> Resposta / Respostas
- Pending Items -> Pendências
- Validation -> Validação
- Readiness -> Prontidão
- Dashboard -> Painel
- Reminder -> Lembrete
- Escalation -> Escalonamento
- Gate -> Critério de Liberação
- Finalize -> Finalizar
- Save -> Salvar
- Save and close -> Salvar e fechar
- View logs -> Ver logs
- Download -> Baixar
- Evidence -> Evidência
- Owner -> Responsável

## Mensagens padrão
- Sucesso: "Resposta salva localmente. Nenhuma alteração foi feita no NetBox."
- Alerta: "Ainda existem pendências antes de avançar."
- Bloqueio: "Operação bloqueada por segurança."
- Risco: "Aprovação humana ainda é obrigatória."

## Campos do modal
- updated_by -> Responsável pelo preenchimento
- owner -> Responsável
- evidence -> Evidência
- notes -> Observações
- tenant -> Cliente/Tenant
- relation_type -> Tipo de relação
- service_relation -> Relação de serviço

## Nota operacional
O texto deve ajudar o operador a concluir a tarefa sem exigir leitura de documentação extra.
