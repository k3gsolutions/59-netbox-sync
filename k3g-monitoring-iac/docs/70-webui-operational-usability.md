# Web UI Operational Usability

## Objetivo
Deixar a Web UI mais direta para operação diária.

## Melhorias
- Próximo passo visível em Pendências de Serviço
- Painel de validação com chamadas claras para validar e finalizar
- Modal com confirmação de status e próximo passo
- Card de Execução Real da Semana 1 no painel
- Termos mais claros em PT-BR

## Termos preferidos
- Pendências de Serviço
- Validação das Respostas
- Comunicação com Times
- Registros de Aprovação
- Logs
- Operação

## Termos que viram linguagem simples
- UAT -> teste controlado
- Gate -> critério de liberação
- Snapshot -> retrato do status
- Pipeline -> validação local
- Draft -> rascunho

## Limites
- Sem escrita NetBox
- Sem apply
- Sem /sync
- Sem aprovação automática
- Revisão humana obrigatória
